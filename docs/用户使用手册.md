## 1. 概述

`giants-swagger` 是一个基于 [Springfox 3.0.0](https://springfox.github.io/springfox/) 的增强扩展，以 Spring Boot Starter 的形式提供。引入依赖后无需编写额外的初始化代码，框架会通过 Spring Boot 的自动装配机制加载相关配置类。

它的目标是让生成的接口文档 **更贴近真实的业务协议、更简洁易读**：

- 将统一的响应封装对象（如 `JsonResult`）自动套用到每一个接口的返回值上；
- 从文档中排除掉与业务无关的参数类型和模型属性（如 `HttpSession`、审计字段等）。

---

## 2. 解决的问题

在使用 Springfox 原生功能生成接口文档时，通常会遇到以下痛点：

| 痛点 | 说明 |
| --- | --- |
| **返回值与真实协议不一致** | 后端一般会用统一的封装对象（如 `JsonResult<T>`）包裹业务数据，但 Controller 方法签名往往直接返回业务对象 `T`。文档里看到的是 `T`，与前端实际收到的 `{ code, message, data }` 结构不符。 |
| **无关参数污染文档** | 像 `HttpSession`、`Cookie`、登录态对象（`LoginEmployee`）等会被 Springfox 当作请求参数展开，出现在文档里，干扰阅读。 |
| **模型冗余字段过多** | 实体对象里的审计字段（`createTime`、`createUserId`、逻辑删除标记等）对调用方没有意义，却全部暴露在模型定义中。 |

`giants-swagger` 针对上述问题分别提供了配置项，做到 **零侵入**（不改动业务代码，仅通过配置文件控制）。

---

## 3. 核心特性

- **统一响应封装**：为所有接口的返回值自动套上封装对象，业务数据放入指定的属性（如 `data`）中。
- **忽略参数类型（全局）**：指定的类型在请求入参和返回结果中都不会出现。
- **忽略请求参数类型**：指定的类型仅在请求入参中被忽略，不会被展开成查询参数。
- **忽略模型属性**：从所有模型定义中移除指定名称的属性。
- **自动装配**：基于 `spring.factories`，引入依赖即生效，无需手动配置 Bean。
- **兼容原生**：所有增强都是对 Springfox 组件的继承与替换，不影响 Springfox 其他能力。

## 5. 快速开始

### 5.1 引入 Maven 依赖

```xml
<dependency>
  <groupId>com.github.vencent-lu</groupId>
  <artifactId>giants-swagger</artifactId>
  <version>2.3.2</version>
</dependency>
```

### 5.2 保证 Springfox 已启用

`giants-swagger` 是对 Springfox 的增强，项目中需要有可用的 Springfox 文档分组配置（`Docket`）。若尚未配置，可参考以下最简示例：

```java
@Configuration
public class SwaggerConfig {

    @Bean
    public Docket docket() {
        return new Docket(DocumentationType.OAS_30)
                .select()
                .apis(RequestHandlerSelectors.basePackage("com.example.controller"))
                .paths(PathSelectors.any())
                .build();
    }
}
```

### 5.3 添加 giants-swagger 配置

在 `application.yml` 中加入配置（各项均为可选，按需开启）：

```yaml
giants:
  swagger:
    return-result-class:
      type: com.giants.web.springmvc.json.JsonResult
      data-property: data
    ignore-parameter-types:
      - javax.servlet.http.HttpSession
      - javax.servlet.http.Cookie
    ignore-request-parameter-types:
      - com.giants.auth.user.domain.LoginEmployee
      - javax.servlet.http.HttpSession
    ignore-model-property-names:
      - delete
      - createUserId
      - createTime
      - lastUpdateUserId
      - lastUpdateTime
```

启动项目，访问 Swagger UI（通常为 `http://localhost:8080/swagger-ui/index.html`），即可看到增强后的文档效果。

## 6. 配置项详解

所有配置均以 `giants.swagger` 为前缀，支持 `yaml` 与 `properties` 两种格式。

### 6.1 return-result-class —— 返回结果封装对象

将所有接口的返回值统一包裹在一个封装对象中，业务数据置于指定属性下。

| 子项 | 类型 | 说明 |
| --- | --- | --- |
| `type` | Class | 封装对象的全限定类名，如 `com.giants.web.springmvc.json.JsonResult`。 |
| `data-property` | String | 封装对象中承载业务数据的属性名，如 `data`。 |

**效果**：假设某接口原本返回 `User`，配置后文档中该接口的返回值会变为：

```json
{
  "code": 0,
  "message": "success",
  "data": { "...User 的字段..." }
}
```

其中 `code`、`message` 等来自封装对象 `JsonResult` 自身的字段，`data` 属性会被替换为接口的真实返回类型。

**properties 写法**：

```properties
giants.swagger.return-result-class.type=com.giants.web.springmvc.json.JsonResult
giants.swagger.return-result-class.data-property=data
```

### 6.2 ignore-parameter-types —— 忽略参数类型（全局）

指定的类型在 **请求入参和返回结果** 中都会被忽略。适用于框架级别、与业务无关的类型。

```yaml
giants:
  swagger:
    ignore-parameter-types:
      - javax.servlet.http.HttpSession
      - javax.servlet.http.Cookie
```

> 该列表会被追加到 Springfox 的 `Defaults.defaultIgnorableParameterTypes()` 中，因此其作用范围与 Springfox 内置的忽略类型一致（全局生效）。

### 6.3 ignore-request-parameter-types —— 忽略请求参数类型

指定的类型仅在 **请求入参** 中被忽略。与 `ignore-parameter-types` 的区别在于：它专门用于拦截 Springfox 对模型属性的展开逻辑，避免某个对象被拆解成一堆查询参数。

```yaml
giants:
  swagger:
    ignore-request-parameter-types:
      - com.giants.auth.user.domain.LoginEmployee
      - javax.servlet.http.HttpSession
```

典型场景：Controller 方法通过参数注入了当前登录用户 `LoginEmployee`，但该对象由框架自动填充，不应作为请求参数出现在文档中。

### 6.4 ignore-model-property-names —— 忽略模型属性名称

从 **所有模型定义** 中移除指定名称的属性。常用于隐藏审计字段、逻辑删除标记等对调用方无意义的字段。

```yaml
giants:
  swagger:
    ignore-model-property-names:
      - delete
      - createUserId
      - createTime
      - lastUpdateUserId
      - lastUpdateTime
```

> 注意：该配置按 **属性名称** 匹配，会作用于文档中出现的每一个模型。因此若不同模型存在同名但含义不同的属性，也会被一并移除，配置时需留意命名冲突。

### 6.5 配置项速查表

| 配置项 | 作用范围 | 匹配方式 | 典型用途 |
| --- | --- | --- | --- |
| `return-result-class` | 返回结果 | 类型 | 统一响应封装 |
| `ignore-parameter-types` | 入参 + 返回 | 类型 | 忽略框架级无关类型 |
| `ignore-request-parameter-types` | 仅入参 | 类型 | 忽略登录态等注入对象 |
| `ignore-model-property-names` | 所有模型 | 属性名 | 隐藏审计/内部字段 |

## 7. 工作原理

`giants-swagger` 通过继承并替换 Springfox 的核心组件来实现增强，各组件职责如下：

### 7.1 自动装配入口

`spring.factories` 声明了 `GiantsSwaggerConfiguration` 为自动配置类：

- `@EnableConfigurationProperties(GiantsSwaggerProperties.class)`：绑定 `giants.swagger` 前缀的配置。
- `@ComponentScan("com.giants.swagger")`：扫描并注册框架内的组件。
- `addIgnorableParameterTypes(...)`：将 `ignore-parameter-types` 追加到 Springfox 的 `Defaults` 忽略列表。

### 7.2 组件替换

`GiantsSwaggerBeanDefinitionRegistryPostProcessor` 在 Bean 定义注册阶段，移除 Springfox 的默认实现并注册本框架的子类：

| 原始 Bean | 替换为 | 作用 |
| --- | --- | --- |
| `serviceModelToSwagger2MapperImpl` | `com.giants.swagger.mappers.ServiceModelToSwagger2MapperImpl` | 处理返回值封装、模型属性剔除 |
| `modelAttributeParameterExpander` | `com.giants.swagger.readers.parameter.ModelAttributeParameterExpander` | 处理请求参数类型忽略 |

### 7.3 返回值封装流程

1. `GiantsCustomModelsProvider`（`OperationModelsProviderPlugin`，最高优先级）在扫描接口时，将封装对象类型注册为已知模型，确保它出现在 `definitions` 中。
2. `ServiceModelToSwagger2MapperImpl.mapDocumentation` 从 `definitions` 中取出封装对象模型 `resultModel`。
3. 遍历所有路径的 `GET/PUT/POST/DELETE` 操作，对每个响应：克隆一份封装对象模型，将其 `data-property` 属性替换为该接口原本的返回类型，再写回响应 schema。

### 7.4 属性与参数忽略流程

- **模型属性忽略**：`mapDocumentation` 遍历所有 `definitions`，逐个移除 `ignore-model-property-names` 中列出的属性。
- **请求参数忽略**：`ModelAttributeParameterExpander.expand` 在展开参数前判断参数类型是否在 `ignore-request-parameter-types` 中，若是则返回空列表，跳过展开。

## 8. 完整示例

以一个用户查询接口为例，演示各配置项的综合效果。

### 8.1 封装对象

```java
public class JsonResult<T> {
    private int code;
    private String message;
    private T data;
    // getter / setter ...
}
```

### 8.2 业务实体

```java
public class User {
    private Long id;
    private String username;

    // 以下审计字段希望从文档中隐藏
    private Boolean delete;
    private Long createUserId;
    private Date createTime;
    private Long lastUpdateUserId;
    private Date lastUpdateTime;
    // getter / setter ...
}
```

### 8.3 Controller

```java
@RestController
@RequestMapping("/users")
public class UserController {

    // LoginEmployee 由框架注入，不应出现在文档参数中
    @GetMapping("/{id}")
    public JsonResult<User> getUser(@PathVariable Long id, LoginEmployee loginEmployee) {
        // ...
    }
}
```

### 8.4 配置

```yaml
giants:
  swagger:
    return-result-class:
      type: com.giants.web.springmvc.json.JsonResult
      data-property: data
    ignore-request-parameter-types:
      - com.giants.auth.user.domain.LoginEmployee
    ignore-model-property-names:
      - delete
      - createUserId
      - createTime
      - lastUpdateUserId
      - lastUpdateTime
```

### 8.5 文档呈现效果

- **请求参数**：只显示 `id`，`loginEmployee` 被忽略。
- **返回结果**：显示为 `JsonResult` 结构，`data` 属性为 `User`。
- **User 模型**：只保留 `id`、`username`，审计字段被移除。

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "username": "zhangsan"
  }
}
```

## 9. 常见问题 FAQ

**Q1：引入依赖后配置不生效？**
请确认：① 项目已正确启用 Springfox（存在可用的 `Docket`）；② 未手动排除 `giants-swagger` 的自动装配；③ 配置前缀为 `giants.swagger`，且类型配置填写的是全限定类名。

**Q2：`ignore-parameter-types` 和 `ignore-request-parameter-types` 有什么区别？**
`ignore-parameter-types` 作用于入参和返回结果（全局），追加到 Springfox 内置忽略列表；`ignore-request-parameter-types` 仅作用于请求入参，用于阻止对象被展开成查询参数。若某类型只想在入参中隐藏，用后者；若在任何位置都不希望出现，用前者。

**Q3：配置了 `return-result-class` 但返回值没有被包裹？**
封装对象需要能被 Springfox 扫描到并生成到 `definitions` 中。请确保封装对象类可被文档扫描到（通常作为某个接口的返回类型出现即可，本框架的 `GiantsCustomModelsProvider` 会主动注册该类型）。同时确认 `type` 填写的类名正确无误。

**Q4：`ignore-model-property-names` 会误删其他模型的同名字段吗？**
会。该配置按属性名称全局匹配，作用于所有模型。若存在同名但语义不同的字段，请谨慎配置或调整字段命名。

**Q5：支持 Springfox 2.x 吗？**
不支持。本框架基于 Springfox 3.0.0（`springfox-boot-starter`）的内部组件实现，需保持该版本。
